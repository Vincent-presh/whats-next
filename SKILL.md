# Whats Next

## What This Does

Turn product context into one sharp next move. Use it when the current task is done and the agent needs to decide what to build next, not just list ideas.

## Inputs

Use real artifacts first:

- product code
- roadmap or backlog
- issue list
- analytics
- support pain
- user feedback

If the ideal product is unclear, define it first in one sentence before ranking anything.

## Process

1. Read the current product state from real artifacts.
1. State the ideal product in one short sentence.
1. Find the biggest gap that matters now.
1. Rank the next 3 features by leverage, dependency, and effort.
1. Pick one feature to build next.
1. If useful, turn that choice into a plan or goal.

## Decision Rules

- Prefer the feature that unlocks more work.
- Prefer the feature that removes a bottleneck.
- Prefer the feature that is small enough to finish soon.
- Prefer the feature that improves the core loop over a shiny side path.
- Do not build a long backlog when one clear next move exists.

See [references/decision-rubric.md](references/decision-rubric.md) for the scoring rubric.

## Output Format

Keep it short:

- what is true now
- what the ideal state is
- what to build next
- why that comes first

If needed, add:

- the top 3 candidates
- one dependency note
- one plan or goal the decision should create

## When To Use

Use after finishing a task, when asking "what should we build next?", or when translating product intent into a concrete follow-up plan.
