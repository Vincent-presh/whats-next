# Decision Rubric

Use this when choosing the next feature.

## Inputs

- current product state
- ideal product state
- known user pain
- blocked dependencies
- effort estimate

## Ranking

Score each candidate feature on:

1. user impact
1. leverage on the roadmap
1. dependency unlocks
1. speed to ship
1. risk if delayed

Prefer the feature with the best total payoff, not the biggest idea.

## Decision rule

- Pick one feature if it clearly wins.
- Pick a small sequence if feature B depends on feature A.
- Stop at 3 candidates unless the product is unclear.

## Good next feature

A good next feature usually:

- removes a bottleneck
- unlocks more work
- improves the core loop
- reduces support or manual work

## Bad next feature

Avoid features that are:

- nice to have but low leverage
- large but speculative
- dependent on unproven future work
- unrelated to the current product gap
